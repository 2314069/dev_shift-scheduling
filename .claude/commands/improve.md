# /improve - システム改善コマンド

## 目的

マルチエージェントシステム自体を継続的に改善するワークフローを実行します。
Architect エージェントが主導し、開発体験の向上を図ります。

## モード

### 1. 振り返りモード
```
/improve 振り返り
```
直近の開発を振り返り、学んだことをシステムに反映。

### 2. 指定改善モード
```
/improve [改善したい点]
```
特定の改善を実行。

## ワークフロー（振り返りモード）

```
Phase 1: 情報収集 → Phase 2: 分析 → Phase 3: 改善計画 → Phase 4: 実行 → Phase 5: 検証
```

## 改善対象

- **CLAUDE.md**: プロジェクト固有の知識、学んだ教訓
- **エージェント定義**（.claude/agents/）: プロンプト改善、責務境界の明確化
- **コマンド**（.claude/commands/）: ワークフローの効率化
- **スキル**（.claude/skills/）: 再利用可能な知識のパッケージ化

## 振り返りのフレームワーク

1. **Keep** - うまくいったこと
2. **Problem** - 問題があったこと
3. **Try** - 試してみたいこと

## 関連コマンド

- `/develop` - 機能開発フロー
- `/fix` - バグ修正フロー
- `/refactor` - リファクタリングフロー
